# gtm-research

Research toolkit for analyzing go-to-market models. Clone, `cd` in, type `claude`, and start researching.

## Quick start

```
/research-full          # Full research sprint on a project
/research-full enclave  # Target a specific project by name
```

## Skills

- `/research-full` — **Full research sprint.** 9-step process: pulls prior context, gathers input, runs competitive + channel research (parallelized), produces validation plan, synthesis doc (NotebookLM-ready), interactive playground, and action brief.
- `/research-init` — Set research context (run first, or let /research-full handle it)
- `/research-social` — Social channel analysis (standalone, or handled by /research-full)
- `/research-competitive` — Competitive landscape mapping (standalone, or handled by /research-full)
- `/research-playground` — Generate interactive HTML dashboard from completed reports

## Project structure

Each project lives in its own directory under `projects/`:

```
projects/
  <project-name>/
    context.md       # Research context (generated by /research-init)
    input/           # Source docs (transcripts, notes, briefs)
    reports/         # Working docs (task list, process log, landscape research)
    output/          # Final outputs (synthesis, playground, action brief)
```

Skills auto-detect projects: if there's only one they use it, if there are multiple they ask which one, if there are none they create one.

## Conventions

- Always cite sources when making claims
- Use WebSearch and WebFetch for research
- Distinguish between hard data and inferences
- Note confidence levels on estimates
