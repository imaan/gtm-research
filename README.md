# gtm-research

Research any go-to-market model using Claude Code.

AI-powered research skills that help you understand how companies go to market — drops, subscriptions, community-led growth, product-led growth, marketplaces, and more — and whether those models could work for your business.

## Quick start

```bash
git clone <repo-url>
cd gtm-research
```

Open in Claude Code, then:

```
/research-full          # Full research sprint on a project
/research-full enclave  # Target a specific project by name
```

That's it. The skill walks you through everything — gathering context, running competitive and channel research in parallel, and producing a synthesis doc, interactive playground, and action brief.

## Skills

| Skill | What it does |
|-------|-------------|
| `/research-full` | **Full research sprint.** 9-step process: gathers context, runs competitive + channel research (parallelized), produces validation plan, synthesis doc (NotebookLM-ready), interactive playground, and action brief. |
| `/research-init` | Set research context — GTM model, category, reference companies. Run standalone or let `/research-full` handle it. |
| `/research-social` | Social channel analysis — community dynamics, engagement patterns, consumer sentiment. |
| `/research-competitive` | Competitive landscape mapping — operators across categories, mechanics, transferable tactics. |
| `/research-playground` | Generate an interactive HTML dashboard to explore findings visually. |

## Project structure

Each project lives in its own directory under `projects/`:

```
projects/
  <project-name>/
    context.md       # Research context (generated by /research-init)
    input/           # Source docs (transcripts, notes, briefs)
    reports/         # Working docs (task list, process log, landscape research)
    output/          # Final outputs (synthesis, playground, action brief)
```

Skills auto-detect projects: if there's only one they use it, if there are multiple they ask which one, if there are none they create one.

## How it works

The skills are **generic** — they adapt at runtime based on your research context. When you describe what GTM model you're studying and for what category, the skills identify the right channels, extraction dimensions, and analysis frameworks for your research.

No templates to customize, no code to generate. Just describe what you're researching and the tools adapt.

## Supported GTM models

Works well for researching:

- **Drops/scarcity** — limited releases, time-gated access, hype-driven
- **Subscriptions** — recurring revenue, retention mechanics, tier structures
- **Community-led growth** — organic advocacy, user communities, network effects
- **Product-led growth** — self-serve, viral loops, freemium-to-paid
- **Marketplace** — two-sided platforms, supply/demand dynamics
- **Freemium** — feature gating, upgrade triggers, usage-based
- **Direct/referral** — word-of-mouth, referral programs, ambassador models

And others — the skills adapt to whatever model you describe.
